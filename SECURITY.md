# Security Policy

## Supported Versions

The following versions of Ophiron currently receive security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| 0.x     | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Ophiron, please report it privately:

- **Do not** open a public GitHub issue.
- Email your report to:  
  **info@glitchidea.com**

Your report should ideally include:

- A detailed description of the vulnerability  
- Potential impact (e.g., RCE, privilege escalation, information disclosure)  
- Steps to reproduce  
- Proof of Concept (if available)

### Response Timeline

- Initial acknowledgment will be sent **within 48 hours**  
- The issue will be reviewed and validated or rejected **within 7 days**  
- For confirmed vulnerabilities, a patch plan and timeline will be shared  
- You will receive status updates throughout the process

### Disclosure Policy

- Valid vulnerabilities remain private until a fix is released  
- A security advisory and/or CVE may be issued depending on severity  
- After a patch is deployed, details will be disclosed responsibly

Thank you for helping keep Ophiron secure.
